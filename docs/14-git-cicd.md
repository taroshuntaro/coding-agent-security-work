# 14 Git・CI/CD・本番操作

[← 目次へ戻る](README.md)

## 14.1 推奨フロー

```text
エージェント
  ↓
作業ブランチ内の変更
  ↓
開発者によるdiff確認
  ↓
テスト・静的解析・シークレットスキャン
  ↓
Pull Request
  ↓
人間のレビュー・承認
  ↓
CI/CD
  ↓
環境ごとの承認ゲート
  ↓
デプロイ
```

## 14.2 エージェントへ与えない権限

[00 R2](00-red-lines.md)の具体化である。

- 保護ブランチへの直接push
- PR承認
- 本番デプロイ承認
- 本番シークレット取得
- 本番DBマイグレーション適用
- IAM・RBACの変更
- 監査ログ削除
- セキュリティ設定の無効化

## 14.3 Git運用

- エージェント作業前に作業ツリーをクリーンにする
- タスクごとにbranchまたはworktreeを分ける
- 小さい変更単位にする
- 自動生成ファイルを含めてdiffを確認
- `.git`、Hooks、CI定義、IDE・エージェント設定の変更を重点確認
- force push、history rewriteを制限
- コミット・PRのAI利用表示方針を組織で決める
