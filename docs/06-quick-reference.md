# 06 レベル別早見表

[← 目次へ戻る](README.md)

本章はレベル×統制の俯瞰である。レベルの決め方は[05 業務利用レベル](05-levels.md)、製品ごとの具体手段は[10 Codex](10-codex.md)・[11 Claude Code](11-claude-code.md)を参照する。

## 6.1 レベル別 統制早見表

| 制御 | L0 | L1 | L2 | L3 | L4 |
|---|---|---|---|---|---|
| 業務利用 | 不可 | 低リスク限定 | 標準 | 機密対応 | 個別審査 |
| ホスト直接実行 | 無制限 | read-only中心 | 原則避ける | 不可を推奨 | 不可 |
| コンテナ | 任意 | 推奨 | 原則利用（同等統制で例外可） | 強化・管理 | VM等と多層化 |
| ワークスペース書込 | 無制限 | 必要時 | 可 | 可・範囲限定 | タスク単位 |
| ワークスペース外 | 無制限 | 原則不可 | deny | deny | deny |
| 承認バイパス | 可 | 不可 | 不可 | 管理的に禁止 | 原則禁止 |
| ネットワーク | 無制限 | 原則無効 | 許可先限定 | 組織プロキシ | deny by default |
| 本番シークレット | あり得る | 禁止 | 禁止 | 禁止 | 原則到達不能 |
| MCP任意追加 | 可 | 原則不可 | 許可リスト | 管理者のみ | 原則無効 |
| 人間のdiffレビュー | 任意 | 必須 | 必須 | 複数者を検討 | 複数者必須 |
| 本番デプロイ | 直接可 | 禁止 | CI/CD | 承認付きCI/CD | 職務分離 |

## 6.2 レベル×プラン 達成手段の早見表

同じレベルでも、統制を**どの手段で実装するか**はプランで異なる。**統制目標（遮断対象）は共通**である。

| レベル | 個人系プランでの達成手段 | チーム・ビジネス系プランでの達成手段 |
|---|---|---|
| L1 | ローカル設定＋使い捨て環境。自己規律＋受入テスト | 同左（管理設定で補強可） |
| L2 | 非rootコンテナ＋ローカル設定＋ネットワーク許可リスト。設定解除は自己規律で抑止 | 管理設定で bypass/full access を**解除不能に固定**。組織監査ログ |
| L3 | 専用VM／強化コンテナ＋OS・MDM・ネットワークプロキシで外部から固定。**製品内強制は不可**のため境界で代替。第三者強制が要件の案件は実施しない | managed requirements／managed settings で**強制**。MCP・Hooks許可リスト。集中監査 |
| L4 | 原則として個人系プランでは要件を満たしにくい。実施するなら使い捨てVM＋DLP＋集中監査を組織基盤として別途用意 | 上記強制＋使い捨てVM＋DLP＋職務分離＋集中監査 |

> [!NOTE]
> 「個人系プランでは製品内強制を作れない」とは、**利用者本人がアカウント設定をいつでも解除できる**ことを指す。外部境界（VM・OS・ネットワーク）は利用者が容易に解除できないため、個人系プランの上位レベルはこの外部境界に依存する。詳細は[README「軸1：契約プラン」](README.md)。

## 6.3 製品強制機能とプランの対応（概要）

| 強制手段 | 個人系 | チーム・ビジネス系 |
|---|---|---|
| Codex managed requirements（`requirements.toml`） | × | ○ |
| Claude Code managed settings（`managed-settings.json`） | × | ○ |
| 組織監査ログ | × | ○ |
| MCP allowlist の管理者強制 | × | ○ |
| ローカル `config.toml` ／ `settings.json` | ○ | ○ |
| コンテナ・VM・OS/MDM・ネットワークプロキシ（製品外） | ○ | ○ |

正確なプラン名・対応状況は変動するため[付録C](appendix-c-volatile-values.md)で確認する。「×」のプランで上位レベルを実施する場合は、製品外の手段（最右行）で代替する。
