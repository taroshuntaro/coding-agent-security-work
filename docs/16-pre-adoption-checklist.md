# 16 導入前チェックリスト

[← 目次へ戻る](README.md)

## 16.0 プラン・レベル

- [ ] 案件のレベルを2軸（機密性×到達範囲）で判定したか（[05](05-levels.md)）
- [ ] 自組織の契約プラン（個人系／チーム・ビジネス系）を確認したか
- [ ] そのレベル×プランで必要な強制が実装可能か確認したか（[06](06-quick-reference.md)）
- [ ] 第三者強制が要件で、個人系プランのため満たせない場合の対応（プラン変更／案件辞退）を決めたか

## 16.1 契約・データ

- [ ] 利用する契約・プランは組織承認済みか
- [ ] 顧客コードを入力・送信してよい契約条件か
- [ ] 学習利用、データ保持、保存場所、リージョンを確認したか
- [ ] 顧客への説明・承諾が必要か
- [ ] 個人アカウント利用を禁止したか（[00 R1](00-red-lines.md)）
- [ ] 退職・異動時のアクセス停止手順があるか

## 16.2 実行環境

- [ ] 対象レベルを決定したか
- [ ] エージェントがどこで動くか明確か
- [ ] コンテナ・VMの境界内にエージェントとコマンド実行環境があるか
- [ ] 非rootか
- [ ] `--privileged`、host network、Docker socket共有がないか
- [ ] ホストへマウントする範囲が対象プロジェクトだけか
- [ ] Dev Container、Dockerfile、Compose、起動スクリプトをレビューしたか
- [ ] IDE拡張・エディタ側プロセスの権限を別途評価したか
- [ ] リソース上限が必要か

## 16.3 権限

- [ ] 読み取り・書き込み範囲を定義したか
- [ ] ワークスペース外をdenyしたか
- [ ] full access・bypassを管理的に禁止したか（[00 R3](00-red-lines.md)）
- [ ] 自動許可・都度確認・拒否コマンドを定義したか
- [ ] 設定をユーザーやリポジトリから解除できないか
- [ ] 管理設定の適用状態を確認する手順があるか
- [ ] 管理設定の取得失敗時にfail-open / fail-closedのどちらになるか確認したか

## 16.4 ネットワーク

- [ ] モデル通信とコマンド通信を分けているか
- [ ] Codexの`disabled` / `cached` / `live`と、Claude Codeの`WebSearch` / `WebFetch`の方針を分けて決めたか
- [ ] 外向き通信はdeny by defaultか
- [ ] 必要ドメインを棚卸ししたか
- [ ] 本番API・DB・管理画面への到達を防いだか
- [ ] プロキシ・DNS・HTTPメソッド制御を検討したか

## 16.5 シークレット

- [ ] 本番資格情報が存在しないか（[00 R2](00-red-lines.md)）
- [ ] `.env`、SSH、クラウド、Kubernetes資格情報を共有していないか
- [ ] 開発専用・最小権限・短寿命か
- [ ] Secret ManagerまたはWorkload Identityを利用できないか
- [ ] ログマスキングがあるか
- [ ] シークレットスキャンがあるか
- [ ] 漏えい時の失効・ローテーション手順があるか
- [ ] 会話履歴、メモリ、ログ、キャッシュの保持期間と削除方法を定義したか

## 16.6 外部連携

- [ ] MCP・Plugins・Hooksを許可リスト化したか
- [ ] 各連携の権限と通信先を確認したか
- [ ] 本番書き込み権限がないか
- [ ] 更新・無効化・監査方法があるか
- [ ] 指示・拡張ファイル（`CLAUDE.md`・`.claude/rules/`・スキル・サブエージェント定義・出力スタイル）をコードレビュー対象にしたか（[2.7](02-terms-and-control-layers.md)・[11.9](11-claude-code.md)）
- [ ] 出力スタイル／システムプロンプト追記で既定の安全指示を上書きしていないか
- [ ] サブエージェント経由のツール実行も権限・サンドボックス・外側境界で限定されることを受入テストで確認したか

## 16.7 開発プロセス

- [ ] Git branch・worktreeを利用するか
- [ ] 人間のdiffレビューを必須化したか（[00 R4](00-red-lines.md)）
- [ ] テスト・Lint・型チェックをCIで再実行するか
- [ ] 依存関係・ライセンス・脆弱性を検査するか
- [ ] 本番デプロイを独立した承認付きCI/CDへ分離したか
- [ ] AI生成コードの責任主体を明確にしたか

## 16.8 実効性確認

- [ ] [15 受入テスト](15-acceptance-tests.md)を対象OS・バージョン・プランで実施したか
- [ ] 設定キーが実際に効いていることを確認したか（[00 R5](00-red-lines.md)）
