# 18 導入推奨パターン

[← 目次へ戻る](README.md)

各パターンは、対応する達成手段がプランで異なる（[06.2](06-quick-reference.md)）。

## 18.1 一般的な社内・顧客開発（レベル2）

```text
レベル2
+ 非root開発コンテナ
+ 対象リポジトリだけbind mount
+ ワークスペース外deny
+ .env / secrets deny
+ ネットワーク許可リスト
+ bypass/full access禁止
+ 開発専用短寿命認証
+ MCP許可リスト
+ 人間のdiffレビュー
+ CI/CD経由のデプロイ
```

- **チーム・ビジネス系**: bypass禁止・MCP許可リストを管理設定で強制。
- **個人系**: 同設定をローカルに置き、外部境界（コンテナ・ネットワーク）と受入テストで補強。

## 18.2 機密性の高い顧客案件（レベル3）

```text
レベル3
+ 組織管理の専用VMまたはクラウド開発環境
+ 案件専用コンテナ・Volume
+ 管理者強制設定
+ 組織プロキシ
+ 組み込みWeb機能を無効または許可先限定
+ Workload Identity / Secret Manager
+ 任意MCP・Hooks禁止
+ 依存取得先の限定
+ 集中監査
+ 複数者レビュー
```

- **チーム・ビジネス系**: 「管理者強制設定」「集中監査」を製品機能で実装。
- **個人系**: 製品内強制が無いため専用VM・OS/MDM・プロキシで境界を作る。第三者強制が契約要件の場合は実施しない（[00 R3・R6](00-red-lines.md)）。

## 18.3 高機密案件（レベル4）

```text
レベル4
+ 利用可否の個別審査
+ タスク単位の使い捨てVM
+ deny by defaultネットワーク
+ 本番到達不能
+ コード・データのローカル保存禁止
+ 動的シークレット
+ DLP・監査・職務分離
+ 承認付きCI/CD
```

必要な管理基盤を持たない場合は実施しない（[05.5](05-levels.md)）。
