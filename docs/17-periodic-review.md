# 17 運用開始後の定期レビュー

[← 目次へ戻る](README.md)

## 17.1 毎月または四半期

- 管理設定と実際の適用状態
- full access・bypass・例外利用
- 新規MCP・プラグイン・Hooks
- ネットワーク許可ドメイン
- 認証情報と権限
- 利用者・所属・案件
- 利用ログと異常操作
- シークレット検出
- エージェント・コンテナイメージ・主要依存の更新

## 17.2 製品更新時

- 新しい権限モード・ツール・外部連携が自動的に有効にならないか
- 設定キーの廃止・既定値変更
- サンドボックスの対応OS・制約
- 管理設定が古いクライアントで無視されないか（Claude Code は `requiredMinimumVersion` で範囲外起動を拒否できる。Codex は全台の対応バージョンを別途確認）
- ネットワーク要件の変更
- データ取扱条件の変更

> [!NOTE]
> 製品更新時は[付録C 揮発値一覧](appendix-c-volatile-values.md)を再確認し、確認日を更新する。あわせて[15 受入テスト](15-acceptance-tests.md)を再実行し、更新前と同じ拒否結果が維持されることを確認する。

## 17.3 インシデント時

1. エージェント・トークン・MCPを停止
2. 到達可能だったシークレットを失効
3. Git、クラウド、ネットワーク、エージェントのログを保全
4. 変更ファイル、外部送信、外部操作を確認
5. 影響範囲を特定
6. シークレットをローテーション
7. ポリシー・設定・教育を是正
8. 顧客・法務・セキュリティへの報告要否を判断
